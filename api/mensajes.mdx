## Overview

The Mensajes API allows you to manage contact form messages in the MTB Backend system. Each message includes sender information, content, submission date, and response status.

<Note>
This endpoint supports draft and publish functionality. Use the `publishedAt` field to control publication status.
</Note>

---

## List All Mensajes

<api-endpoint method="GET" path="/api/mensajes" />

Retrieve a list of all contact messages with optional filtering, sorting, and pagination.

### Query Parameters

<ParamField query="filters" type="object" optional>
  Filter messages by field values. Example: `filters[respondido][$eq]=false`
</ParamField>

<ParamField query="sort" type="string" optional>
  Sort messages by field(s). Example: `fecha:desc` for newest first
</ParamField>

<ParamField query="pagination[page]" type="number" optional default="1">
  Page number for pagination
</ParamField>

<ParamField query="pagination[pageSize]" type="number" optional default="25">
  Number of items per page
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of mensaje objects
  
  <Expandable title="Mensaje Object">
    <ResponseField name="id" type="number">
      Unique identifier for the message
    </ResponseField>
    
    <ResponseField name="attributes" type="object">
      <Expandable title="Attributes">
        <ResponseField name="nombre" type="string" required>
          Name of the message sender
        </ResponseField>
        
        <ResponseField name="email" type="string" required>
          Email address of the sender
        </ResponseField>
        
        <ResponseField name="mensaje" type="array" required>
          Message content in blocks format
        </ResponseField>
        
        <ResponseField name="fecha" type="datetime">
          Date and time when the message was submitted
        </ResponseField>
        
        <ResponseField name="respondido" type="boolean" default="false">
          Whether the message has been responded to
        </ResponseField>
        
        <ResponseField name="publishedAt" type="datetime">
          Publication timestamp (null if draft)
        </ResponseField>
        
        <ResponseField name="createdAt" type="datetime">
          Creation timestamp
        </ResponseField>
        
        <ResponseField name="updatedAt" type="datetime">
          Last update timestamp
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination metadata
  
  <Expandable title="Meta Object">
    <ResponseField name="pagination" type="object">
      <ResponseField name="page" type="number">
        Current page number
      </ResponseField>
      <ResponseField name="pageSize" type="number">
        Number of items per page
      </ResponseField>
      <ResponseField name="pageCount" type="number">
        Total number of pages
      </ResponseField>
      <ResponseField name="total" type="number">
        Total number of messages
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/api/mensajes?filters[respondido][$eq]=false&sort=fecha:desc" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/mensajes?filters[respondido][$eq]=false&sort=fecha:desc', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/api/mensajes',
    params={
        'filters[respondido][$eq]': 'false',
        'sort': 'fecha:desc'
    },
    headers={
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_TOKEN'
    }
)
data = response.json()
```
</CodeGroup>

## Create a Mensaje

<api-endpoint method="POST" path="/api/mensajes" />

Create a new contact message. This is typically used by contact forms on the frontend.

### Request Body

<ParamField body="data" type="object" required>
  <Expandable title="Mensaje Data">
    <ParamField body="nombre" type="string" required>
      Name of the message sender
    </ParamField>
    
    <ParamField body="email" type="string" required>
      Email address of the sender
    </ParamField>
    
    <ParamField body="mensaje" type="array" required>
      Message content in blocks format
    </ParamField>
    
    <ParamField body="fecha" type="datetime" optional>
      Date and time of submission (auto-generated if not provided)
    </ParamField>
    
    <ParamField body="respondido" type="boolean" optional default="false">
      Whether the message has been responded to
    </ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="data" type="object">
  The created mensaje object
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X POST "https://api.example.com/api/mensajes" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "nombre": "Sofía López",
      "email": "sofia.lopez@example.com",
      "mensaje": [
        {
          "type": "paragraph",
          "children": [{"type": "text", "text": "Me gustaría inscribirme en la próxima carrera. ¿Cuáles son los requisitos?"}]
        }
      ],
      "fecha": "2024-01-25T16:00:00.000Z"
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/mensajes', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    data: {
      nombre: 'Sofía López',
      email: 'sofia.lopez@example.com',
      mensaje: [
        {
          type: 'paragraph',
          children: [{type: 'text', text: 'Me gustaría inscribirme en la próxima carrera. ¿Cuáles son los requisitos?'}]
        }
      ],
      fecha: new Date().toISOString()
    }
  })
});
const data = await response.json();
```

```python Python
import requests
from datetime import datetime

response = requests.post(
    'https://api.example.com/api/mensajes',
    headers={'Content-Type': 'application/json'},
    json={
        'data': {
            'nombre': 'Sofía López',
            'email': 'sofia.lopez@example.com',
            'mensaje': [
                {
                    'type': 'paragraph',
                    'children': [{'type': 'text', 'text': 'Me gustaría inscribirme en la próxima carrera. ¿Cuáles son los requisitos?'}]
                }
            ],
            'fecha': datetime.now().isoformat()
        }
    }
)
data = response.json()
```
</CodeGroup>

## Update a Mensaje

<api-endpoint method="PUT" path="/api/mensajes/:id" />

Update an existing message by its ID. Commonly used to mark messages as responded.

### Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the message to update
</ParamField>

### Request Body

<ParamField body="data" type="object" required>
  Mensaje fields to update (same structure as Create)
</ParamField>

### Response

<ResponseField name="data" type="object">
  The updated mensaje object
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X PUT "https://api.example.com/api/mensajes/1" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "data": {
      "respondido": true
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/mensajes/1', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({
    data: {
      respondido: true
    }
  })
});
const data = await response.json();
```

```python Python
import requests

response = requests.put(
    'https://api.example.com/api/mensajes/1',
    headers={
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_TOKEN'
    },
    json={
        'data': {
            'respondido': True
        }
    }
)
data = response.json()
```
</CodeGroup>

## Message Management Tips

### Filtering Unresponded Messages

To get all messages that haven't been responded to:

```bash
GET /api/mensajes?filters[respondido][$eq]=false&sort=fecha:asc
```

### Marking Messages as Responded

After responding to a message via email or other means, update the message:

```javascript
PUT /api/mensajes/:id
{
  "data": {
    "respondido": true
  }
}
```

### Date Filtering

To get messages from a specific date range:

```bash
GET /api/mensajes?filters[fecha][$gte]=2024-01-01&filters[fecha][$lte]=2024-01-31
```
