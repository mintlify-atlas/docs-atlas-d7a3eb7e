## Overview

The Inscripciones API allows you to manage event registrations in the MTB Backend system. Each registration includes participant information, category, payment details, and Flow payment gateway integration.

<Note>
This endpoint does NOT support draft and publish functionality. All inscripciones are immediately available once created.
</Note>

---

## List All Inscripciones

<api-endpoint method="GET" path="/api/inscripcions" />

Retrieve a list of all event registrations with optional filtering, sorting, and pagination.

### Query Parameters

<ParamField query="filters" type="object" optional>
  Filter registrations by field values. Example: `filters[estadoPago][$eq]=Pagado`
</ParamField>

<ParamField query="sort" type="string" optional>
  Sort registrations by field(s). Example: `createdAt:desc`
</ParamField>

<ParamField query="pagination[page]" type="number" optional default="1">
  Page number for pagination
</ParamField>

<ParamField query="pagination[pageSize]" type="number" optional default="25">
  Number of items per page
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of inscripcion objects
  
  <Expandable title="Inscripcion Object">
    <ResponseField name="id" type="number">
      Unique identifier for the registration
    </ResponseField>
    
    <ResponseField name="attributes" type="object">
      <Expandable title="Attributes">
        <ResponseField name="nombreCompleto" type="string">
          Full name of the participant
        </ResponseField>
        
        <ResponseField name="rut" type="string">
          Chilean national ID (RUT) of the participant
        </ResponseField>
        
        <ResponseField name="edad" type="number">
          Age of the participant
        </ResponseField>
        
        <ResponseField name="categoria" type="string">
          Competition category
        </ResponseField>
        
        <ResponseField name="tipo" type="string">
          Registration type (Open or Federado)
        </ResponseField>
        
        <ResponseField name="email" type="string">
          Email address of the participant
        </ResponseField>
        
        <ResponseField name="telefono" type="string">
          Phone number of the participant
        </ResponseField>
        
        <ResponseField name="monto" type="number">
          Registration amount in local currency
        </ResponseField>
        
        <ResponseField name="tokenFlow" type="string">
          Flow payment gateway token
        </ResponseField>
        
        <ResponseField name="ordenFlow" type="string">
          Flow payment order number
        </ResponseField>
        
        <ResponseField name="estadoPago" type="string">
          Payment status: `Pendiente`, `Pagado`, or `Rechazado`
        </ResponseField>
        
        <ResponseField name="createdAt" type="datetime">
          Creation timestamp
        </ResponseField>
        
        <ResponseField name="updatedAt" type="datetime">
          Last update timestamp
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination metadata
  
  <Expandable title="Meta Object">
    <ResponseField name="pagination" type="object">
      <ResponseField name="page" type="number">
        Current page number
      </ResponseField>
      <ResponseField name="pageSize" type="number">
        Number of items per page
      </ResponseField>
      <ResponseField name="pageCount" type="number">
        Total number of pages
      </ResponseField>
      <ResponseField name="total" type="number">
        Total number of registrations
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/api/inscripcions?filters[estadoPago][$eq]=Pagado&sort=createdAt:desc" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/inscripcions?filters[estadoPago][$eq]=Pagado&sort=createdAt:desc', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/api/inscripcions',
    params={
        'filters[estadoPago][$eq]': 'Pagado',
        'sort': 'createdAt:desc'
    },
    headers={
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_TOKEN'
    }
)
data = response.json()
```
</CodeGroup>

## Create an Inscripcion

<api-endpoint method="POST" path="/api/inscripcions" />

Create a new event registration.

### Request Body

<ParamField body="data" type="object" required>
  <Expandable title="Inscripcion Data">
    <ParamField body="nombreCompleto" type="string" optional>

    <ParamField body="edad" type="number" optional>

    </ParamField>
    
    <ParamField body="categoria" type="string" optional>
      Competition category
    </ParamField>
    
    <ParamField body="tipo" type="string" optional>
      Registration type (e.g., "Open", "Federado")
    </ParamField>
    
    <ParamField body="email" type="string" optional>
      Email address of the participant
    </ParamField>
    
    <ParamField body="telefono" type="string" optional>
      Phone number of the participant
    </ParamField>
    
    <ParamField body="monto" type="number" optional>
      Registration amount in local currency

## Payment Status Values

The `estadoPago` field can have one of the following values:

- **Pendiente** (default): Payment is pending
- **Pagado**: Payment has been successfully processed
- **Rechazado**: Payment was rejected or failed
